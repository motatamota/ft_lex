ft_lex 自体の実装に使用できる言語は以下の通り。
ただし、ft_lex が**出力するコードは C 言語**でなければならない。
すべての言語において**標準ライブラリのみ**使用可能。

| 言語 | 備考 |
|------|------|
| C | |
| C++ | |
| Java | |
| Zig | |
| Rust | |
| Caml/OCaml | |
| 任意の純粋関数型言語 | |
| アセンブリ | 非推奨（"please don't"） |

### その他の制約

- 実行ファイル名: `ft_lex`
- libl のライブラリ名: `libl`
- コンパイル言語を使う場合、Makefile またはネイティブビルドシステムの設定を提出すること
- 出力される C ファイル内で使用可能な関数:
  - `<stdio.h>` の全関数
  - `<string.h>` の全関数
  - `malloc(3)`, `realloc(3)`, `calloc(3)`, `free(3)`

## POSIX lex 正規表現

lex の `.l` ファイル内で使用できる正規表現の構文一覧。

### 基本マッチ

| 構文 | 意味 | 例 |
|------|------|-----|
| `c` | 文字 c そのもの | `a` → `a` にマッチ |
| `.` | 改行以外の任意の 1 文字 | |
| `\n` | 改行文字 | |
| `\t` | タブ文字 | |
| `\\` | バックスラッシュそのもの | |

### 繰り返し（量指定子）

| 構文 | 意味 | 例 |
|------|------|-----|
| `*` | 直前の要素が 0 回以上 | `a*` → `""`, `"a"`, `"aaa"` |
| `+` | 直前の要素が 1 回以上 | `a+` → `"a"`, `"aaa"` (空はNG) |
| `?` | 直前の要素が 0 回か 1 回 | `a?` → `""`, `"a"` |
| `{n}` | ちょうど n 回 | `a{3}` → `"aaa"` |
| `{n,}` | n 回以上 | `a{2,}` → `"aa"`, `"aaa"`, ... |
| `{n,m}` | n 回以上 m 回以下 | `a{2,4}` → `"aa"`, `"aaa"`, `"aaaa"` |

### 文字クラス `[...]`

| 構文 | 意味 | 例 |
|------|------|-----|
| `[abc]` | a, b, c のどれか 1 文字 | |
| `[a-z]` | a から z の範囲のどれか | |
| `[0-9]` | 0 から 9 の範囲のどれか | |
| `[a-zA-Z]` | 英字（大文字小文字） | |
| `[^abc]` | a, b, c **以外**のどれか | |
| `[^0-9]` | 数字**以外**のどれか | |

### POSIX 文字クラス `[[:name:]]`

`[...]` の中で使用する。

| 構文 | 意味 |
|------|------|
| `[[:alpha:]]` | アルファベット (`[a-zA-Z]` 相当) |
| `[[:digit:]]` | 数字 (`[0-9]` 相当) |
| `[[:alnum:]]` | 英数字 (`[a-zA-Z0-9]` 相当) |
| `[[:blank:]]` | スペースとタブ |
| `[[:space:]]` | 空白文字全般 (スペース, タブ, 改行など) |
| `[[:upper:]]` | 大文字 (`[A-Z]` 相当) |
| `[[:lower:]]` | 小文字 (`[a-z]` 相当) |
| `[[:print:]]` | 印字可能文字 |
| `[[:punct:]]` | 句読点・記号 |

### グループと選択

| 構文 | 意味 | 例 |
|------|------|-----|
| `(expr)` | グループ化 | `(ab)+` → `"ab"`, `"abab"`, ... |
| `r\|s` | r または s | `cat\|dog` → `"cat"` or `"dog"` |

### クォートとエスケープ

| 構文 | 意味 | 例 |
|------|------|-----|
| `"..."` | クォート内の特殊文字を無効化（ダブルクォートのみ） | `"+"` → `+` そのものにマッチ |
| `\c` | 特殊文字 c をエスケープ | `\*` → `*` そのものにマッチ |

- シングルクォート `'` は特別な意味を持たない。`'a'` は `'`, `a`, `'` の3文字にマッチするパターンとして扱われる
- `"..."` 内でも `\\` や `\"` は有効（それぞれ `\` と `"` にマッチ）

### エスケープシーケンス一覧

正規表現内およびクォート内で使用できるエスケープ。

| 構文 | 意味 | ASCII（10進） | ASCII（16進） |
|------|------|:---:|:---:|
| `\n` | 改行（newline） | 10 | 0x0A |
| `\t` | タブ（horizontal tab） | 9 | 0x09 |
| `\r` | キャリッジリターン | 13 | 0x0D |
| `\f` | フォームフィード | 12 | 0x0C |
| `\v` | 垂直タブ | 11 | 0x0B |
| `\a` | ベル | 7 | 0x07 |
| `\\` | バックスラッシュそのもの | 92 | 0x5C |
| `\"` | ダブルクォートそのもの（クォート内で使用） | 34 | 0x22 |
| `\0` | NUL 文字 | 0 | 0x00 |
| `\nnn` | 8進数 nnn で指定した文字（例: `\101` → `A`） | — | — |
| `\xhh` | 16進数 hh で指定した文字（例: `\x41` → `A`） | — | — |
| `\c` | 上記以外の文字 c はそのもの（メタ文字の無効化） | — | — |

#### 文字クラス `[...]` 内の空白

文字クラス内では空白やタブもエスケープなしで並べられる：

```lex
[ \t]    → スペースまたはタブ
[ \t\n]  → スペース、タブ、改行のどれか
```

クォート外（文字クラス外）の空白はパターンとアクションの**区切り**として解釈されるため、リテラルの空白が必要なときは `" "` または `[ ]` を使う。

### アンカーと先読み

| 構文 | 意味 | 例 |
|------|------|-----|
| `^` | 行頭にマッチ | `^hello` → 行頭の `hello` のみ |
| `$` | 行末にマッチ | `world$` → 行末の `world` のみ |
| `r/s` | r にマッチ（後ろに s が続く場合のみ） | `abc/def` → `"abcdef"` の中の `"abc"` |

## マッチングルール

lex が入力を処理する際の優先ルール:

1. **最長一致 (Longest Match)**: 複数ルールがマッチ可能な場合、最も長くマッチするものを選ぶ
2. **先書き優先 (First Match)**: 同じ長さでマッチした場合、`.l` ファイルで先に記述されたルールが優先
3. **デフォルト動作 (ECHO)**: どのルールにもマッチしない文字は、そのまま標準出力に出力される

## `.l` ファイルの構造

```
定義セクション（名前付きパターン、Cコードの%{...%}ブロック）
%%
ルールセクション（正規表現 + アクション）
%%
ユーザーコードセクション（そのまま出力される追加Cコード）
```

### 名前定義で使用できる文字

名前（識別子）は C の識別子と同じルール：

```
[A-Za-z_][A-Za-z0-9_]*
```

| 種類 | 使用可否 | 例 |
|------|:-------:|----|
| 英字（大文字・小文字） | ✓ | `DIGIT`, `letter` |
| アンダースコア | ✓ | `_DIGIT`, `my_token` |
| 数字（2文字目以降） | ✓ | `TOKEN2` |
| 数字（先頭） | ✗ | `2DIGIT` |
| ハイフン | ✗ | `MY-TOKEN` |
| 空白 | ✗ | `MY TOKEN`（名前と定義の区切りとして解釈） |
| 記号（`.` `+` `*` など） | ✗ | `MY.TOKEN` |

名前定義は**必ず1行**。複数行にまたがる構文は存在しない。

### 定義セクションの例

```
%{
#include <stdio.h>
%}

DIGIT   [0-9]
LETTER  [a-zA-Z]
```

### ルールセクションの例

```
%%
{DIGIT}+          printf("NUMBER: %s\n", yytext);
{LETTER}+         printf("WORD: %s\n", yytext);
"+"               printf("PLUS\n");
\n                /* 改行を無視 */
.                 printf("Invalid character: %c\n", *yytext);
```

### 主要な変数・関数

| 名前 | 説明 |
|------|------|
| `yytext` | 現在マッチしたテキスト (`char *`) |
| `yyleng` | マッチしたテキストの長さ (`int`) |
| `yyin` | 入力ファイル (`FILE *`, デフォルトは `stdin`) |
| `yyout` | 出力ファイル (`FILE *`, デフォルトは `stdout`) |
| `yylex()` | レキサーのメイン関数 |
| `yywrap()` | 入力終了時に呼ばれる関数（libl が提供） |
| `ECHO` | マッチした文字列をそのまま出力するマクロ |
| `input()` | 入力から次の 1 文字を読む |
| `unput(c)` | 1 文字を入力に戻す |

## コマンドラインオプション（POSIX 必須）

POSIX（IEEE Std 1003.1-2024）が `lex` ユーティリティに要求する必須オプションは **3つ**。

| オプション | 説明 |
|----------|------|
| `-n` | `-v` で出力される統計サマリを抑制する（デフォルト動作と同じ） |
| `-t` | 生成 C コードを `lex.yy.c` ではなく**標準出力**に書き出す |
| `-v` | テーブルサイズなどの**統計サマリ**を標準出力に出力（`-t` と併用時は stderr へ） |

デフォルト出力ファイルは `lex.yy.c`。引数なしまたは `-` で stdin から読む。

## 12.2 Utility Syntax Guidelines

POSIX Base Definitions の Section 12.2 で定められた、**コマンドライン引数の統一ルール**。`lex` を含む全 POSIX ユーティリティが従う。

| ガイドライン | 内容 |
|------------|------|
| オプションは `-` で始まる | 1文字の英数字。例: `-t`, `-v` |
| オプションの結合 | `-t -v` は `-tv` と書ける |
| オプションの終端 `--` | `--` 以降はオプションではなくオペランドとして扱う |
| オプション引数 | オプションが引数を取る場合 `-o file` または `-ofile` のどちらでも可 |
| オペランドはオプションの後 | `ft_lex -tv input.l` のようにオプションを先に書く |
| `-` 単独 | ファイル名の代わりに使うと stdin を意味する |

`getopt` を使うとこれらのルールを自動的に満たせる：

```cpp
while ((opt = getopt(argc, argv, "ntv")) != -1)
{
    switch (opt)
    {
        case 'n': options.no_summary = true; break;
        case 't': options.to_stdout  = true; break;
        case 'v': options.verbose    = true; break;
    }
}
// argv[optind] 以降がオペランド（入力ファイル）
```
